SOP：以 Cloudflare Tunnel + Access 對外發布前台／後台

家用 Mac Mini 零對外 port 對外服務，後台子網域套用白名單 Gmail 存取控制。

方案：Cloudflare Tunnel（cloudflared，outbound）＋ Cloudflare Access（邊緣身分驗證）。

建立日期 2026-07-05 子網域區分 Token 託管（A1） 不需 port forwarding
⚠ 2026-07-05 更新（本頁部分內容已過時）：改為單一網域＋路徑分流。 兩層子網域（如 admin.e-learning.example.com）超出 Universal SSL 憑證涵蓋範圍，TLS 會握手失敗。 現行做法：Public Hostname 只掛一條一層子網域 → http://proxy:80（repo 新增的 nginx 反代）， /admin/* → web-admin、API 路徑 → api、其餘 → web-learner； Access Application 以「網域＋Path admin」保護後台。 詳見同名 Markdown 版（已更新）。

0背景與原則▶
使用者瀏覽器 │ https://admin.example.com / https://learn.example.com ▼ Cloudflare 邊緣 ├─ Access 政策（只在 admin.*：白名單 Gmail）→ 注入 Cf-Access-Jwt-Assertion ▼ （outbound tunnel，家裡不開任何 port） cloudflared 容器（Mac Mini 內） ├─ admin.example.com → web-admin:80 └─ learn.example.com → web-learner:80 ▼ web-admin / web-learner（nginx，不動） └─ /articles /words /audio … 反代 → api:8080（同源，不變）
  • 兩個前端映像、nginx、api、worker 全部維持不動；本 SOP 只是「在最前面加一層對外入口」。
  • api 不對外；前端 nginx 內部反代 /articles、/words、/audio… 到 api:8080 照舊。
  • 邊緣注入的 Cf-Access-Jwt-Assertion 一路穿過 cloudflared → nginx → api，沿用現有 CF_ACCESS_AUD / CF_ACCESS_TEAM_DOMAIN / ADMIN_EMAILS 驗證鏈。
  • Tunnel 是 Mac Mini 主動 outbound，家用路由器不需開任何 port。
  • 路由與 Access 政策皆採 Dashboard 託管（Token 模式），repo 只多一個容器與一個 .env 變數。
命名慣例：以下用 admin.example.com（後台）、learn.example.com（前台）為例，實作時請替換成你的網域。
1前置需求▶
2程式碼異動（已進 repo）▶
提示：本節異動已在 commit 3f966e8 進版控，正式機 git pull 後即有。以下列出內容供對照。

2.1 docker-compose.yml 新增 cloudflared 服務（tunnel profile）

yaml
  cloudflared:
    image: cloudflare/cloudflared:latest
    profiles: ["tunnel"]
    command: tunnel run
    restart: unless-stopped
    environment:
      TUNNEL_TOKEN: ${CLOUDFLARE_TUNNEL_TOKEN}
    depends_on:
      - web-admin
      - web-learner

2.2 正式機 .env 設定

dotenv
# 前後台互連（build 時注入 SPA）
ADMIN_URL_PUBLIC=https://admin.example.com
LEARNER_URL_PUBLIC=https://learn.example.com

# 關閉本機用的驗證繞道（正式機務必為 0）
DEV_AUTH_BYPASS=0

# Cloudflare Tunnel Token（見第 3 步取得；敏感值，勿進 repo）
CLOUDFLARE_TUNNEL_TOKEN=<在步驟 3 取得>
VITE_ADMIN_URL / VITE_LEARNER_URL 會自動吃到這些值。改了互連網址後前端需重新 build（見第 4 步）。
3Cloudflare 後台一次性設定▶
位置：Cloudflare Dashboard → Zero Trust → 左側選單。

3.1 建立 Tunnel 並取得 Token

  • Networks → Tunnels → Create a tunnel。
  • 類型選 Cloudflared，命名（如 english-learning）。
  • 建立後畫面顯示的安裝指令裡，--token 後那串就是 Tunnel Token → 複製貼到正式機 .env 的 CLOUDFLARE_TUNNEL_TOKEN。（我們不用它的安裝指令，容器會用 Token 自己連。）

3.2 設定 Public Hostname（ingress 路由）

在該 Tunnel 的 Public Hostname 分頁新增兩條：

SubdomainDomainTypeService URL
adminexample.comHTTPweb-admin:80
learnexample.comHTTPweb-learner:80
  • Service URL 直接填容器服務名（cloudflared 與前端在同一 docker network）。
  • Type 用 HTTP（前端 nginx 聽 80；對外 TLS 由 Cloudflare 邊緣負責）。
  • DNS 的 CNAME 由 Cloudflare 自動建立，不需手動加。

3.3 建立 Access Application（後台白名單）

  • Access → Applications → Add an application → Self-hosted。
  • Application name：English Learning Admin。
  • Application domain：admin.example.com（只綁後台）。
  • Policies 新增一條：Action = Allow，Rule = Emails → 你的白名單 Gmail（可多個）。
  • 前台 learn.example.com 不要建 Application → 保持公開。
若之後要讓前台也需登入，再為 learn.example.com 建一個 Application 即可，互不影響。
4部署（正式機 Mac Mini）▶
bash
# 1. 取得最新程式碼與 .env（含 Token、正式子網域、DEV_AUTH_BYPASS=0）
git pull

# 2. 重新 build 前端（互連網址是 build 時注入，改了就要重建）
docker compose build web-admin web-learner

# 3. 啟動所有服務 + tunnel（--profile tunnel 才會帶起 cloudflared）
docker compose --profile tunnel up -d

# 4. 確認 cloudflared 已連上（看到 "Registered tunnel connection" 即成功）
docker compose logs -f cloudflared
本機開發不加 --profile tunnel，cloudflared 不會啟動，開發不受影響。
5驗證（Deploy 後必做）▶
6回滾（Rollback）▶

Tunnel 是「附加層」，回滾很單純：

bash
# 只停掉對外入口，內部服務照跑
docker compose stop cloudflared
  • 停掉 cloudflared 後，子網域會 502／無法連線，但內部服務與資料完全不受影響。
  • 要完全撤除：於 Cloudflare 後台刪除該 Tunnel 與 Access Application，並移除 .env 的 Token。
7疑難排解▶
症狀可能原因處理
子網域回 502cloudflared 連得上，但接不到前端容器確認 Public Hostname 的 Service 填 web-admin:80 / web-learner:80；確認前端容器 healthy
子網域一直轉圈cloudflared 沒起來或 Token 錯docker compose logs cloudflared；檢查 .env 的 Token
後台打 API 得 401/403JWT 沒穿透或 DEV_AUTH_BYPASS 設錯確認正式機 DEV_AUTH_BYPASS=0、CF_ACCESS_AUD/CF_ACCESS_TEAM_DOMAIN/ADMIN_EMAILS 正確
後台不用登入就進得去Access Application 沒綁對 domain確認 domain 為 admin.example.com 且政策 Allow→Emails
互連按鈕跳到 localhost前端沒用正式網址重 build設好 ADMIN_URL_PUBLIC/LEARNER_URL_PUBLIC 後 docker compose build web-admin web-learner
白名單外也進得去政策 Action 設成 Bypass 或規則過寬檢查 Access Policy：Action=Allow、Rule=Emails 精確清單
8日常維運備忘▶
  • 新增／移除白名單 Gmail：Cloudflare 後台改 Access Policy 即可，不需重部署。
  • 改路由目標：Cloudflare 後台改 Public Hostname 即可，不需重部署。
  • cloudflared 升級：docker compose pull cloudflared && docker compose --profile tunnel up -d cloudflared。
  • Token 外洩：後台刪除該 Tunnel 重建、換新 Token、更新 .env 後重啟 cloudflared。
